# Planificación del proyecto

## Objetivo

Documentar colaborativamente una infraestructura LAMP para una PYME utilizando Git y GitHub.

---

# Organización del trabajo

## Alumno A

Responsable de:

- Análisis de requisitos
- Diseño de infraestructura
- Servidor web Apache
- Base de datos MySQL

## Alumno B

Responsable de:

- SSH y Firewall
- Monitorización
- Backups
- Operación y recuperación

---

# Flujo de trabajo Git

Se utilizará el siguiente flujo:

1. Crear rama feature
2. Realizar cambios
3. Hacer commits descriptivos
4. Subir rama a GitHub
5. Crear Pull Request
6. Revisar cambios del compañero
7. Resolver conflictos si aparecen
8. Fusionar en main

---

# Cronograma

| Sesión | Trabajo |
|--------|----------|
| Sesión 1 | Creación repositorio y estructura |
| Sesión 2 | Desarrollo de documentación |
| Sesión 3 | Resolución de conflictos |
| Sesión 4 | Revisión final y release |

---

# Herramientas utilizadas

- Git
- GitHub
- Markdown
- PowerShell
- Ubuntu Server 22.04

---

# Riesgos identificados

- Conflictos entre ramas
- Cambios simultáneos
- Errores en merges
- Desactualización de ramas locales

---

# Medidas preventivas

- Uso frecuente de git pull
- Pull Requests obligatorios
- Revisiones cruzadas
- Commits pequeños y descriptivos
